# SLAOps Documentation

SLAOps is a DevOps platform for monitoring, logging, and analysing HTTP API usage across your applications — helping you track SLA compliance, diagnose issues, and control API spend.

This section is the **customer knowledge base**. Everything here is written for teams evaluating or using SLAOps.

## What you can do with SLAOps

- **Monitor SLAs** — track response time, error rate, and availability per API endpoint, with alerts when thresholds are breached
- **Troubleshoot issues** — search structured HTTP request/response logs enriched with OpenAPI operation metadata
- **Track spend** — understand the cost and volume of every API your services call

## How to get started

1. [Getting Started](./getting-started) — connect your first application via the relay
2. [OASpec Bucket](./oaspec-bucket) — upload your OpenAPI specs to enrich your logs
3. [Multi-Tenancy](./multi-tenancy) — understand how your data is isolated
